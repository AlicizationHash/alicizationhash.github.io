1.尝试中的一体化解决方案—-WhateverGreen

1.1-什么是WhateverGreen

WhateverGreen 是 Github 上的一个开源项目，它旨在于使得 Hackintosh 更加容易的驱动。原文 ( [Guide] Intel Framebuffer patching using WhateverGreen ) 中提到，它已经整合了WhateverGreen, IntelGraphicsFixup, NvidiaGraphicsFixup, Shiki，CoreDisplayFixup这些Kext，这是以前 Hackintosh 中要驱动显卡的必要 Kext ，仍有一些 Kext 没有整合进去，相信它会在将来完善得更好

1.2-Lilu+WhateverGreen

WhateverGreen需要Lilu扩展库才可使用，使用前请确保你的Kext文件夹中有且是最新版的Lilu

1.3-WhateverGreen的使用方法

正如上面所提到的，WhateverGreen已经整合了以上驱动，所以如果将这些驱动放入你的 kext 文件夹中的话将会引起冲突。
原文中也提到了，你需要执行以下操作:

  1. Remove:
  • FakePCIID_Intel_HD_Graphics
  • IntelGraphicsFixup
  • NvidiaGraphicsFixup
  • CoreDisplayFixup
  • Shiki
  1. Turn off all graphics injections in Clover:
  • config.plist/Graphics/Inject/ATI=NO
  • config.plist/Graphics/Inject/Intel=NO
  • config.plist/Graphics/Inject/NVidia=NO
  • config.plist/Graphics/ig-platform-id=
  • config.plist/Devices/FakeID/IntelGFX=
  1. Disable DSDT Clover fixes:
  • AddHDMI
  • FixDisplay
  • FixIntelGfx
  • AddIMEI
  1. Disable UseIntelHDMI
  2. Remove boot argument: -disablegfxfirmware
  3. Remove any IGPU and HDMI entries from:
  • config.plist/Devices/Arbitrary
  • config.plist/Devices/Properties
  • config.plist/Devices/AddProperties
  1. Remove any IGPU and HDMI related SSDT and DSDT from:
  • CLOVER/ACPI/patched
  1. Renaming GFX0 -> IGPU
  • WhateverGreen will do this automatically (see caveat below)
  • Be aware that WhateverGreen does not rename all instances of GFX0 -> IGPU but should be okay in most cases
  • You may need to include Clover GFX0 -> IGPU rename for other kexts or ACPI patching that require it
    一般来说的话，第8步是不需要进行的，大部分的情况中它们工作的很好，但是如果-v图中出现了一些带有IGPU字样的提示并且进入系统之后没有驱动显卡，那你就要执行第8步，虽然原文中提到 “WhateverGreen will do this automatically”，但是WhateverGreen不会重命名 GFX0 -> IGPU 的所有实例，你就需要ACPI Patching，在这里我们只讨论正常情况，恕不再多说
    执行完成上述操作后，你就可以把WhateverGreen和Lilu放入你的Kext文件夹使用了。

1.4-可能会遇到的问题–DVMT问题

1.4.1-什么是DVMT问题

当今世界上的一些游戏或者软件，都需要显示内存到达制定标准才可以启动，但是因为笔记本电脑本身的限制，显存不可能分配的太多(针对以前的电脑，现代计算机显存完全够用，但是这项技术仍然存在)，于是，动态分配共享显存技术(Dynamic Video Memory Technology,简称DVMT)应运而生。

”所分配的视频内存量取决于操作系统要求的内存量。当不再需要视频内存时，将其返还操作系统供其它应用程序或系统功能使用。DVMT 根据系统需求分配内存。有些较新的系统在系统 BIOS 中有一个选项，用于调整可用于 DVMT 的内存容量。动态分配的内存最高可达图形驱动程序设置的最大值。 –百度百科DVMT词条

这便是DVMT技术，它给Windows用户带来了很大的便利，但是给Hackintosh造成了很大的麻烦，在 MacOS X Yosemite (10.10+) 版本中，Apple Inc.的开发人员提升了Stolen Memory的最小值，使得任何BIOS设置中Pre-allocated Memory值小于66MB的计算机出现Kernel Panic即内核崩溃。在笔记本的BIOS中，多数是64MB的默认值且不存在设置Pre-allocated Memory值的选项，所以这就导致了”DVMT问题”

1.4.2-我该如何应对

很幸运的，有巨佬注意到了这个问题，IntelGraphicsDVMTFixup.kext应运而生(WhateverGreen中并没有集成这个驱动，需要手动添加)，这个驱动使得DVMT问题迎刃而解，不再是问题
对于另一些BIOS里可以调整Pre-allocated Memory值的电脑，只需要将其设置为128MB即可解决问题

2.传统的方式，Inject Intel

2.1-注意事项

这种传统的方式只适用于macOS High Sierra及以下版本，在macOS Mojave中，Apple Inc.改变了注入的方式，使得Inject Intel需要配合Clover的KextsToPatch使用，操作比较(麻)神(烦)奇所以不在讨论范围之内

2.2-原理

其实上面说的所有这些，不管是WhateverGreen还是Inject Intel，它们并不是一个完整的显卡驱动，显卡驱动Apple Inc.已经写好了并且放入了系统中，它们所做的一切，仅仅是让macOS识别你的显卡并且加载相应的驱动，这也是早先为什么能有DSDT驱动法等等，如果macOS识别不到正确的显卡ID是不会加载驱动的，Clover的Inject Intel正是起到了这个作用，它”伪装”你的显卡ID，然后”告诉”macOS：我的显卡就是这个ID，请加载驱动，然后…macOS它就会相信你的显卡确实是正确的ID，然后加载驱动，你就成功的驱动了显卡

2.3-使用方法

由于Inject Intel会与WhateverGreen冲突，你会需要删除WhateverGreen然后才能使用Inject Intel
它非常的简单，只需要几步操作(以我的HD5500为例子):
1.打开你的config.plist文件
2.找到:Graphics/Inject/Intel，把它的String改成ture
3.找到:Graphcis/Inject/ig-platform-id，把它的String改成:0x16160002(仅适用于我的HD5500，其他显卡请自己寻找ID)
4.保存并退出
5.Restart and Enjoy !

2.4遇到的问题

多半都是DVMT问题，参照上方一样的DVMT解决方案
补充:还有一种是花屏，这时候你需要放入IntelGraphicsFixup.kext来解决花屏

3.如果我的电脑使用Chameleon引导该怎么办–Inject Intel-ig

3.1 写在前面

虽然现在Clover的时代，并没有太多的人使用Chameleon，很不幸的，我的笔记本对UEFI的支持不完整，Clover EFI引导会出现无解问题，模拟UEFi屡遭失败，只能使用Chameleon来进行引导，此方法已经在Chameleon r2908版本测试成功，其他版本请自行测试，已知这个版本可以驱动三代(Ivy Bridge)、四代(Haswell)、五代(Broadwell)核显，测试显卡为 Intel(R) HD Graphics 5500

写在最后

其实Hackintosh并没有想象中的那么难，主要受硬件限制，Intel平台的相较于AMD平台的来说安装相对容易，这些东西也并非什么普通人掌握不了的东西，我们都是从普通人过来的，相信自己，万物皆有可能

驱动了显卡，你就又迈出了一步，或许你还有声卡和网卡没有驱动，那攻克它们就是你的下一步，声卡可以利用AppleALC仿冒驱动或者VoodooHDA万能驱动，如果你的网卡很不幸的无解，你可以选择给你的笔记本或者台式机换一张网卡，建议选购Apple的原装网卡，这样利于驱动
